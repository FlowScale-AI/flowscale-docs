## Deployment Options

### REST APIs

Convert workflows into HTTP endpoints that integrate with any application:

<CodeGroup>

```bash curl
curl -X POST https://api.flowscale.ai/v1/run \
  -H "Authorization: Bearer fs_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_id": "text-to-image-v2",
    "inputs": {
      "prompt": "A serene mountain landscape at sunset",
      "width": 1024,
      "height": 1024,
      "steps": 20,
      "cfg_scale": 7.5
    }
  }'
```

```python Python
import flowscale

client = flowscale.Client(api_key="fs_1234567890abcdef")

result = client.run(
    workflow_id="text-to-image-v2",
    inputs={
        "prompt": "A serene mountain landscape at sunset",
        "width": 1024,
        "height": 1024,
        "steps": 20,
        "cfg_scale": 7.5
    }
)

print(result.images[0].url)
```

```javascript JavaScript
import FlowScale from '@flowscale/sdk';

const client = new FlowScale({
  apiKey: 'fs_1234567890abcdef'
});

const result = await client.run({
  workflowId: 'text-to-image-v2',
  inputs: {
    prompt: 'A serene mountain landscape at sunset',
    width: 1024,
    height: 1024,
    steps: 20,
    cfgScale: 7.5
  }
});

console.log(result.images[0].url);
```

</CodeGroup>

## Deployment Workflow

### Simple 4-Step Process

<Steps>
  <Step title="Prepare Workflow">
    Ensure your workflow runs successfully in the workspace and produces the desired outputs
  </Step>
  <Step title="Configure API">
    Define input parameters, validation rules, and output formats
  </Step>
  <Step title="Deploy">
    Click deploy and choose your scaling and security settings
  </Step>
  <Step title="Test & Share">
    Use the generated documentation to test your API and share with users
  </Step>
</Steps>

### Configuration Options

<Tabs>
  <Tab title="Input Configuration">
    **Parameter Types**:
    - **Text**: Prompts, descriptions, filenames
    - **Number**: Steps, CFG scale, dimensions
    - **Image**: Upload widgets with format validation
    - **Boolean**: Toggles for optional features
    - **Select**: Dropdown menus for predefined options

    **Validation Rules**:
    - Min/max values for numbers
    - Character limits for text
    - File size and format restrictions
    - Required vs. optional parameters
  </Tab>
  <Tab title="Output Configuration">
    **Response Formats**:
    - **Images**: URLs to generated images with metadata
    - **JSON**: Structured data with custom fields
    - **Files**: Direct download links for any file type
    - **Streaming**: Real-time updates for long-running workflows

    **Content Delivery**:
    - Global CDN for fast image delivery
    - Temporary URLs with configurable expiration
    - Permanent storage options for important outputs
    - Compression and optimization settings
  </Tab>
  <Tab title="Performance Settings">
    **GPU Selection**:
    - **T4**: Basic workflows, development testing
    - **L4**: Standard image generation, moderate complexity
    - **A100**: Complex workflows, batch processing
    - **H100**: Largest models, real-time applications

    **Scaling Configuration**:
    - Min/max instance counts
    - Auto-scaling triggers and thresholds
    - Cold-start optimization settings
    - Geographic distribution preferences
  </Tab>
</Tabs>

## Integration Examples

### Common Use Cases

<Tabs>
  <Tab title="Web Applications">
    **Frontend Integration**:
    ```javascript
    // React component example
    const [image, setImage] = useState(null);
    const [loading, setLoading] = useState(false);
    
    const generateImage = async (prompt) => {
      setLoading(true);
      try {
        const result = await flowscale.run({
          workflowId: 'text-to-image',
          inputs: { prompt }
        });
        setImage(result.images[0].url);
      } catch (error) {
        console.error('Generation failed:', error);
      }
      setLoading(false);
    };
    ```
  </Tab>
  <Tab title="Mobile Applications">
    **Flutter/React Native**:
    ```dart
    // Flutter example
    Future<String> generateImage(String prompt) async {
      final response = await http.post(
        Uri.parse('https://api.flowscale.ai/v1/run'),
        headers: {
          'Authorization': 'Bearer $apiKey',
          'Content-Type': 'application/json',
        },
        body: jsonEncode({
          'workflow_id': 'text-to-image',
          'inputs': {'prompt': prompt}
        }),
      );
      
      final data = jsonDecode(response.body);
      return data['images'][0]['url'];
    }
    ```
  </Tab>
  <Tab title="Backend Services">
    **Microservice Integration**:
    ```python
    # Python service example
    from fastapi import FastAPI
    import flowscale
    
    app = FastAPI()
    client = flowscale.Client(api_key=os.getenv('FLOWSCALE_API_KEY'))
    
    @app.post("/generate")
    async def generate_content(request: GenerateRequest):
        result = await client.run(
            workflow_id="content-generator",
            inputs=request.dict()
        )
        return {"content": result.outputs}
    ```
  </Tab>
</Tabs>
