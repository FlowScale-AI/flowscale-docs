# One-Click Shareable UI for Non-Technical Collaborators

You've configured a deployment with proper schema validation and access controls. Now you need to **let stakeholders try it** without requiring technical knowledge, API keys, or code.

Playground UIs solve this by auto-generating **parameter-bounded interfaces** from your deployment configuration, with **guardrails**, **presets**, and **shareable links** that make AI workflows accessible to creative reviewers, product managers, and business stakeholders.

<Callout type="info">
**For AI & Creative Leads:** This page shows how to create safe sandbox environments for stakeholder collaboration.
**For Developers:** See [Custom Playground Integration](#custom-playground-integration) for embedding options.
**For Leadership:** Review [Usage Governance](#usage-governance) for compliance and audit capabilities.
</Callout>

---

## When to Use Playground UIs

| Scenario | Playground Benefits | Alternative Approaches |
|---|---|---|
| **Creative Review Cycles** | Non-technical teams can iterate on prompts, styles, and parameters safely | Developers manually generating variations |
| **Stakeholder Demos** | Self-service exploration with preset bounds during meetings | Static screenshots or recorded demos |
| **Client Presentations** | Branded, professional interface that clients can interact with | Sharing raw API endpoints or technical docs |
| **User Acceptance Testing** | Product teams can validate AI behavior before development integration | Building custom test interfaces |
| **Training & Onboarding** | New team members learn workflow capabilities hands-on | Documentation-only learning |

### Parameter Controls Based on Schema

FlowScale AI automatically creates appropriate UI controls based on your deployment schema:

<Tabs>
  <Tab title="Text Inputs">
    **Schema Definition:**
    ```typescript
    {
      "prompt": {
        "type": "string",
        "maxLength": 1000,
        "description": "Describe the image you want to create"
      }
    }
    ```
    
    **Generated UI:**
    - Large text area with character counter (0/1000)
    - Placeholder text from description
    - Real-time validation feedback
    - Suggested prompts dropdown (if configured)
  </Tab>
  <Tab title="Numeric Sliders">
    **Schema Definition:**
    ```typescript
    {
      "creativity": {
        "type": "number", 
        "minimum": 0.1,
        "maximum": 2.0,
        "default": 0.7,
        "step": 0.1,
        "description": "Higher values = more creative, unexpected results"
      }
    }
    ```
    
    **Generated UI:**
    - Slider with min/max bounds (0.1 ↔ 2.0)  
    - Current value display (0.7)
    - Helpful labels: "Conservative ← → Creative"
    - Tooltip with description on hover
  </Tab>
  <Tab title="Dropdown Selections">
    **Schema Definition:**
    ```typescript
    {
      "style_preset": {
        "type": "string",
        "enum": ["photographic", "digital_art", "comic_book", "fantasy_art"],
        "default": "photographic",
        "description": "Visual style for the generated image"
      }
    }
    ```
    
    **Generated UI:**
    - Dropdown with enum options
    - Preview thumbnails for each style (if configured)
    - Default selection highlighted
    - Search/filter for long lists
  </Tab>
  <Tab title="Locked Parameters">
    **Schema Definition:**
    ```typescript
    {
      "model_checkpoint": {
        "type": "string",
        "enum": ["stable-diffusion-xl"],
        "default": "stable-diffusion-xl", 
        "locked": true,
        "description": "Model version (locked for brand consistency)"
      }
    }
    ```
    
    **Generated UI:**
    - Read-only display: "Model: Stable Diffusion XL"
    - Lock icon indicating parameter is fixed
    - Tooltip explaining why it's locked
    - Hidden from UI entirely (if configured)
  </Tab>
</Tabs>

<CardGroup cols={2}>
  <Card title="Quick Start Presets" icon="zap">

<CodeGroup>
```yaml Preset Configuration
presets:
  - name: "Product Photography"
    description: "Clean, professional product shots"
    parameters:
      prompt: "professional product photography, white background, studio lighting"
      style_preset: "photographic"
      creativity: 0.3
      aspect_ratio: "1:1"
      
  - name: "Social Media"
    description: "Eye-catching posts for Instagram/Facebook" 
    parameters:
      prompt: "vibrant, engaging social media post"
      style_preset: "digital_art"
      creativity: 0.8
      aspect_ratio: "4:5"
```

```javascript Custom Preset API
// Teams can save their own presets programmatically
const customPreset = await playground.savePreset({
  name: "Brand Campaign Q4",
  description: "Holiday season marketing materials",
  parameters: {
    prompt: brandGuidelines.holidayPromptTemplate,
    style_preset: "fantasy_art", 
    creativity: 0.6,
    color_palette: ["#FF4444", "#00AA44"] // Brand colors
  },
  sharing: "team" // or "private", "organization"
});
```
</CodeGroup>

## Shareable Links & Access Control

### Link Types & Permissions

<Tabs>
  <Tab title="Public Share Links">
    **Use Case:** Client presentations, external collaborator reviews
    
    ```
    https://playground.flowscale.ai/share/abc123?expires=2024-02-15
    ```
    
    **Features:**
    - No login required
    - Configurable expiration (1 day → 6 months)
    - Usage limits (10 generations, 100 API calls)
    - Optional password protection
    - Watermarked outputs (if configured)
    
    **Security:** Rate limited, no API key exposure, audit logged
  </Tab>
  <Tab title="Team-Only Links">
    **Use Case:** Internal reviews, stakeholder feedback sessions
    
    ```
    https://playground.flowscale.ai/team/creative-services/workflow-v2
    ```
    
    **Features:**
    - SSO authentication required
    - Full parameter access (within bounds)
    - Usage tracking by individual user
    - Team-wide preset sharing
    - Integration with Slack/Teams notifications
    
    **Security:** Organization auth, role-based access, full audit trails
  </Tab>
  <Tab title="Embedded Playgrounds">
    **Use Case:** Custom applications, internal tools integration
    
    ```html
    <iframe 
      src="https://playground.flowscale.ai/embed/workflow-123"
      data-auth-token="embed_token_xyz"
      width="800" height="600">
    </iframe>
    ```
    
    **Features:**
    - Customizable branding and styling
    - Parent application authentication
    - PostMessage API for result handling
    - Responsive design for mobile/desktop
    - White-label options available
    
    **Security:** CORS controls, token-based auth, sandbox isolation
  </Tab>
</Tabs>

### Link Configuration Example

<CodeGroup>
```yaml Client Demo Link
link_type: "public_share"
permissions:
  max_generations: 20
  time_limit: "4_hours"  # Demo duration
  parameter_restrictions:
    - lock: ["model_checkpoint", "brand_guidelines"]
    - limit: { creativity: { max: 0.8 } }  # Prevent extreme outputs
branding:
  logo: "company_logo.png"
  primary_color: "#0066CC"
  title: "New Product Line Concept Generator"
watermark: 
  enabled: true
  text: "Confidential Preview - Company Name"
```

```yaml Team Review Link  
link_type: "team_authenticated"
permissions:
  max_concurrent_users: 10
  session_duration: "8_hours" 
  full_parameter_access: true
  preset_creation: true
integrations:
  slack_notifications:
    channel: "#creative-reviews"
    notify_on: ["generation_complete", "preset_saved"]
  usage_tracking:
    track_individual_usage: true
    export_reports: true
```
</CodeGroup>

## Playground Templates by Use Case

### Creative Agency Template

<CodeGroup>
```yaml Client Review Setup
name: "client-review-playground"
description: "Safe client-facing creative review environment"

ui_config:
  layout: "presentation_mode"
  parameter_visibility: "essential_only"  # Hide technical parameters
  presets:
    - "Brand Guidelines Compliant"
    - "Campaign Theme A" 
    - "Campaign Theme B"
    - "Alternative Direction"

guardrails:
  content_filter: "strict"  # No potentially inappropriate content
  brand_compliance: "enforce"  # Must match brand guidelines
  output_quality: "high"  # No low-quality/glitched results

client_controls:
  parameter_access: "preset_only"  # Clients can only use presets
  download_permissions: "watermarked"
  sharing: "disabled"  # Prevent client from sharing outside
  
session_config:
  max_duration: "2_hours"  # Presentation meeting length
  max_generations: 50
  auto_save_favorites: true
```

```yaml Internal Team Setup
name: "creative-team-sandbox"
description: "Full-featured playground for internal creative iteration"

ui_config:
  layout: "power_user"
  parameter_visibility: "all"
  advanced_controls: true
  preset_creation: true
  
collaboration:
  real_time_sharing: true  # Multiple users can work simultaneously  
  comment_system: true
  version_branching: true  # Save alternative directions
  
integrations:
  figma_export: true
  slack_notifications: true
  asset_library_sync: true

session_config:
  max_duration: "unlimited"
  max_generations: 500  
  auto_save_frequency: "every_5_generations"
```
</CodeGroup>
