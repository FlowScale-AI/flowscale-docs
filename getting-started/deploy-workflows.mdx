## Deployment Options

### API Endpoints

Transform workflows into REST APIs that can be integrated into any application:

<CodeGroup>

```bash curl
curl -X POST https://api.flowscale.ai/v1/run \
  -H "Authorization: Bearer your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "workflow_id": "wf_abc123",
    "inputs": {
      "prompt": "A beautiful sunset over mountains",
      "steps": 20,
      "cfg": 7.5
    }
  }'
```

```javascript JavaScript
const response = await fetch('https://api.flowscale.ai/v1/run', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your-api-key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    workflow_id: 'wf_abc123',
    inputs: {
      prompt: 'A beautiful sunset over mountains',
      steps: 20,
      cfg: 7.5
    }
  })
});
```

```python Python
import requests

response = requests.post(
    'https://api.flowscale.ai/v1/run',
    headers={
        'Authorization': 'Bearer your-api-key',
        'Content-Type': 'application/json'
    },
    json={
        'workflow_id': 'wf_abc123',
        'inputs': {
            'prompt': 'A beautiful sunset over mountains',
            'steps': 20,
            'cfg': 7.5
        }
    }
)
```

</CodeGroup>

## Configuration Options

### Input/Output Settings

Configure how your workflow interacts with the API:

<Tabs>
  <Tab title="Input Parameters">
    - **Text prompts**: String inputs with validation
    - **Numeric values**: Sliders, number inputs with ranges
    - **Image uploads**: File handling with format restrictions
    - **Dropdown options**: Predefined choices for models/styles
  </Tab>
  <Tab title="Output Formats">
    - **Images**: PNG, JPEG, WebP with quality settings
    - **JSON data**: Structured response with metadata
    - **File downloads**: Direct file URLs for generated content
    - **Batch results**: Multiple outputs from single request
  </Tab>
</Tabs>

### GPU & Scaling Settings

Choose the right compute resources for your workflow:

| GPU Type | Use Case | Performance | Cost |
|----------|----------|-------------|------|
| **T4** | Simple workflows, testing | Basic | $ |
| **L4** | Standard image generation | Good | $$ |
| **A100** | Complex workflows, batch processing | Excellent | $$$ |
| **H100** | Largest models, real-time inference | Premium | $$$$ |

### Auto-scaling Configuration

- **Concurrency limits**: Maximum simultaneous requests
- **Cold start optimization**: Faster response times
- **Geographic regions**: Deploy closer to your users
- **Timeout settings**: Request processing limits

## Production Considerations

### Security & Authentication

- **API keys**: Secure access with regeneratable keys
- **Rate limiting**: Protect against abuse and manage costs
- **CORS configuration**: Control cross-origin access
- **Webhook support**: Real-time notifications for long-running jobs

### Performance Optimization

<AccordionGroup>
  <Accordion title="Caching Strategies">
    - Model caching to reduce cold start times
    - Result caching for identical requests
    - CDN integration for faster content delivery
  </Accordion>
  <Accordion title="Request Optimization">
    - Async processing for long-running workflows
    - Batch requests to reduce overhead
    - Progressive loading for large outputs
  </Accordion>
  <Accordion title="Resource Management">
    - Auto-scaling based on demand
    - GPU pooling for efficient utilization
    - Memory optimization for large models
  </Accordion>
</AccordionGroup>
