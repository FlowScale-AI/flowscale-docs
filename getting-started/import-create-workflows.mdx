## Bringing Your Breakthrough to the Team

**This is where your local creative success becomes a collaborative asset.** The import process takes your proven workflow and transforms it into something your entire team can access, modify, and deploy without losing any of its power or sophistication.

### Supported Import Format

**We focus on the standard for maximum reliability:** FlowScale AI imports **Workflow JSON** filesâ€”the same format you get from ComfyUI's "Save" button. This ensures your workflow maintains complete fidelity during the transition from local to cloud.

<Warning>
**File Format Matters:** Only use JSON files exported directly from ComfyUI's Save function. Other export formats or manually edited files may cause import issues or lost functionality.
</Warning>

### CPU vs GPU Mode: Smart Resource Management

**This is how you stay creative without breaking budgets:** FlowScale AI separates workflow development from workflow execution, so you only pay for GPU power when you actually need it.

<Tabs>
  <Tab title="CPU Mode (Default)">
    **Your creative sandbox:**
    - **Perfect for**: Connecting nodes, adjusting parameters, organizing your workflow logic
    - **Cost impact**: Minimalâ€”focus on creativity without cost anxiety
    - **Best practices**: Do all your planning, node connections, and parameter tweaking here
    - **Why this matters**: You can spend hours perfecting your workflow structure without burning through GPU credits
  </Tab>
  
  <Tab title="GPU Mode">
    **Your execution powerhouse:**
    - **Perfect for**: Running inference, testing outputs, validating results
    - **Cost impact**: Higher, but you're getting enterprise-grade GPU performance
    - **Best practices**: Switch here only when you need to see actual results
    - **Why this matters**: Access to GPUs that cost thousands to own, available instantly when you need them
  </Tab>
</Tabs>

<Tip>
**Cost Optimization**: Always use CPU mode for editing workflows and only switch to GPU mode when you need to execute them. This prevents unnecessary credit burn.
</Tip>

## Model Management: From Chaos to Control

**Remember managing models locally?** The endless downloads, version conflicts, storage concerns, and "why isn't this model loading?" debugging sessions. FlowScale AI transforms this into **a centralized, team-accessible model library** where everyone shares the same high-quality assets.

<Note>
**Why manual model management is actually better:** Unlike automatic systems that might pull the wrong versions or create conflicts, FlowScale AI gives you complete control over which exact models your workflows use. This ensures consistency across your team and prevents the "works on my machine" problems that plague local ComfyUI setups.
</Note>

### Folder Structure Guidelines

When creating folders for models, follow ComfyUI conventions:

- **Checkpoints**: `models/checkpoints/`
- **LoRAs**: `models/loras/`
- **VAE**: `models/vae/`
- **Custom nodes**: Check the node's GitHub repository for specific folder requirements

<Warning>
For custom node models, always refer to the node's GitHub repository README for correct folder naming conventions.
</Warning>
