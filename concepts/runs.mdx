**Runs** are individual execution records created each time a workflow is executed within a container via Flowscale's ComfyUI API. These records provide a comprehensive log of workflow activity, including input and output data, making it easier to track and analyze each execution.

<img src="https://flowscale-prod.s3.us-east-1.amazonaws.com/docs/Screenshot+2024-11-05+180828.png" alt="API Keys Screenshot"/>


## Key Features

- **Execution Tracking**: Capture detailed information for each run, including inputs, outputs, and metadata.
- **Historical Data**: Maintain a record of past runs to support data analysis, debugging, and performance monitoring.

## How to Use Runs

### 1. Execute a Workflow

- Initiate a workflow run using API calls or the API Playground.
- Each execution generates a new run record with all associated details.

### 2. View Run Details

- Go to the **Runs** tab in your project.
- Click on a specific run to view its details, including timestamps, parameters, and status.

<img src="https://flowscale-prod.s3.us-east-1.amazonaws.com/docs/Screenshot+2024-11-05+181010.png" alt="API Keys Screenshot"/>


### 3. Analyze Results

- Use historical data to identify trends or optimize workflow configurations.

---

With Runs, Flowscale allows users to track and analyze each execution of their workflows, providing valuable data for improving accuracy, troubleshooting, and performance optimization.
