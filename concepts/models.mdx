**Models** refer to key assets like **checkpoints**, **LoRAs**, **ControlNets**, **VAEs**, and others used within ComfyUI workflows. Models are stored in **Volumes**, which can be either public or private, allowing users to share common resources or securely manage private assets. Additionally, models in Flowscale are **project-agnostic**—once added, they are accessible across all projects, enhancing resource management and workflow efficiency.

<img src="https://flowscale-prod.s3.us-east-1.amazonaws.com/docs/Screenshot+2024-11-05+144352.png" alt="API Keys Screenshot"/>

## Key Features

- **Model Types**: Support for various model types, including checkpoints, LoRAs, ControlNets, VAE, and more (see the complete list below).
- **Volume Management**: Organize models within **public** or **private volumes** depending on access requirements.
- **Custom Model Uploads**: Users can add and manage models within private volumes for exclusive use, maintaining security and privacy.
- **Project-Agnostic**: Models are accessible across all projects once uploaded, supporting centralized management.

## How to Use Models

### 1. Accessing Public and Private Volumes

- Navigate to the **Models** section in Flowscale.
- Use **Public Volumes** to view and access shared models available to all users.
- Use **Private Volumes** to manage personal or proprietary models, enabling secure usage across workflows.

### 2. Adding Models to Private Volumes

<img src="https://flowscale-prod.s3.us-east-1.amazonaws.com/docs/Screenshot+2024-11-05+174552.png" alt="API Keys Screenshot"/>

- In the **Private Volumes** section, click on the **+** icon and select **Add Folder** to organize models.
- Within the folder, click the **⋮** icon and choose **Upload Model**.
- Search for the model or enter the download URL, filename(optional), and source credentials(CivitAI API Key or HuggingFace API Key) 
- Click **Upload** to add it to your private volume.

### 3. Using Models in ComfyUI Workflows

- Within a workflow, select the relevant node configuration where models are applied (e.g., checkpoints, ControlNets).
- Choose a model from the volume, adjusting parameters as necessary.
- Integrate the model directly within the workflow, allowing for seamless interaction with other nodes.

## Benefits

- **Access Control**: Public volumes provide shared access, while private volumes ensure secure storage for proprietary models.
- **Workflow Customization**: Customize workflows with specialized models to meet specific project needs.
- **Project-Agnostic Scalability**: Models added once are accessible across all projects, enabling efficient resource sharing and centralized management.
